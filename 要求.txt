
### 1. 产品概述
- **产品名称**：LinkNote
- **定位**：一款支持多媒体输入、以“双向关联”为核心的新式便签工具，帮助用户构建个人知识网络。
- **目标平台**：Web 单页应用（响应式设计，优先移动端体验），后端简单数据服务（可选本地存储模式）。

### 2. 用户故事
1. 用户可以快速创建一张便签，输入文字、语音或图片，甚至混合使用。
2. 用户能在便签之间建立**双向链接**（类似 Roam Research 的 `[[ ]]` 引用），点击链接可跳转到关联便签，并看到反向链接列表。
3. 用户可以在“图谱视图”中看到便签之间的关联网络，自由拖拽节点，直观浏览知识结构。
4. 所有数据本地自动保存，并支持导出为 JSON / Markdown 备份。

### 3. 功能需求详细版

#### 3.1 便签基础管理
- **创建便签**：提供 `+` 按钮，创建空白便签，自动生成唯一 ID 和时间戳。
- **多媒体输入**：
  - **文字**：支持纯文本，可内嵌 `[[便签标题]]` 创建关联。
  - **语音**：提供录音按钮，录制后生成音频条，可播放、暂停、删除。音频保留原文件（WebM/mp3），不自动转文字（减少复杂度）。
  - **图片**：支持粘贴、拖拽、点选上传图片，自动压缩到合理尺寸（如 1200px 宽），以 Base64 或对象 URL 预览，存储时转为 Blob/文件引用。
- **编辑与保存**：
  - 实时自动保存（本地 IndexedDB），关闭页面不丢失。
  - 支持便签标题，标题可编辑，默认为便签创建时间。
- **删除与恢复**：暂不做回收站（MVP 阶段），删除时二次确认。

#### 3.2 关联系统（核心升级点）
- **关联的定义**：
  - 当便签 A 的文字内容中包含 `[[B的标题]]` 时，系统自动建立一条从 A 到 B 的**单向引用**，同时在 B 的下方显示“被以下便签引用”列表，形成双向可见。
- **关联展示**：
  - **反向链接面板**：打开任一便签时，底部/侧边显示所有引用了当前便签的其他便签列表。
  - **图谱总览**：一个独立视图，以力导向图展示所有便签节点和引用连线，节点大小代表引用次数，可全屏拖拽，点击节点跳转到便签。
- **取消关联**：从引用源中删除 `[[ ]]` 即断开，反向链接实时更新。

#### 3.3 搜索与过滤
- 全文搜索便签标题和文本内容（不搜索音频和图片内部）。
- 搜索结果高亮显示，支持按时间排序。

#### 3.4 数据持久化与导出
- **本地优先**：使用 IndexedDB 存储所有便签数据和媒体文件 Blob，不用注册账号。
- **导出**：将所有便签、关联信息、音频/图片文件一起导出为 zip 包，也可单独导出 JSON。
- **导入**：支持导入之前导出的 zip 包恢复数据。

#### 3.5 界面与交互
- **主界面**：左侧便签列表（按时间或字母排序），中间便签编辑区，右侧可选的反向链接面板。
- **便签卡片**：显示标题、内容缩略、多媒体图标（语音/图片小标）。
- **移动端适配**：列表与编辑分屏滑动。

### 4. 非功能需求
- **性能**：便签总数 < 2000 时，列表渲染不卡顿；图谱节点 < 500 时动画流畅。
- **存储**：IndexedDB 容量需监控，单文件上传不超过 10MB。
- **隐私**：所有数据仅存本地，不上传任何服务器（MVP 阶段完全离线可用）。
- **兼容**：支持现代浏览器 Chrome/Edge/Firefox/Safari 最新版。

### 5. 技术栈建议（给 Claude Code 的提示）
- 前端：React + TypeScript + Tailwind CSS
- 状态管理：本地使用 Zustand + IndexedDB wrapper（idb 库）
- 图谱可视化：d3-force 或 cytoscape.js
- 录音：MediaRecorder API
- 图片压缩：browser-image-compression
- 导出：JSZip + FileSaver
- 构建工具：Vite

### 6. 交付物与里程碑
1. 基础便签 CRUD + 文字输入 + 自动保存
2. 语音 & 图片输入模块
3. 关联系统：`[[]]` 解析、反向链接面板
4. 图谱视图
5. 搜索、导出/导入
6. UI 美化与移动端适配
